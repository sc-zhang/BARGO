BARGO (**B**ayesian **A**lignment-based **R**IL **G**ene-level **O**rigin inference) is a lightweight Bayesian framework for fast gene-level inference of parental origin in 
recombinant inbred lines (RILs) using competitive read alignment evidence from dual-reference genomes.

## Introduction
BARGO performs **gene-level parental origin inference** by comparing alignment evidence of sequencing reads mapped to parental reference genomes.

For each $gene_g$, BARGO aggregates alignment evidence under a Bayesian decision framework with a dual-threshold classification system.

## Installation
```bash
git clone https://github.com/sc-zhang/BARGO.git
cd BARGO
pip install -r requirements.txt
chmod +x bargo.py
```

## Usage
```bash
usage: bargo.py [-h] -A A -B B --A_gff3 A_GFF3 --B_gff3 B_GFF3 [--A_cds A_CDS] [--B_cds B_CDS] -l LIST [--labels LABELS] [--tau TAU] [--gamma GAMMA] [--min_mapq MIN_MAPQ] -o OUTPUT [-t THREADS] [--version]

options:
  -h, --help            show this help message and exit
  -A A                  Input bam directory of parent A
  -B B                  Input bam directory of parent B
  --A_gff3 A_GFF3       Input gff3 file of parent A
  --B_gff3 B_GFF3       Input gff3 file of parent B
  --A_cds A_CDS         Input CDS file of parent A
  --B_cds B_CDS         Input CDS file of parent B
  -l, --list LIST       Input homologous gene pair list
  --labels LABELS       Name of parents, comma split, default="A,B"
  --tau TAU             Posterior decision threshold, default=0.7
  --gamma GAMMA         Confidence filtering threshold, default=0.3
  --min_mapq MIN_MAPQ   Minimum mapping quality, default=20
  -o, --output OUTPUT   Output directory of tsv files of classification
  -t, --threads THREADS
                        Number of threads for parallel processing, default=1
  --version             show program's version number and exit
```
> [!WARNING]  
> - Parameters A and B are directories containing BAM files generated by aligning reads from the same set of samples 
> to reference genomes A and B, respectively. BAM files representing the same sample must have identical filenames 
> in both directories, like:
> ```bash
> A--S1.bam
>  |-S2.bam
>  |-S3.bam
> 
> B--S1.bam
>  |-S2.bam
>  |-S3.bam
> ```
> - All BAM files must be indexed.
> - The homologous gene pair list must be a tab-delimited text file with two columns. 
> The first column contains gene IDs from reference A, while the second column contains the corresponding homologous 
> gene IDs from reference B. An example is shown below:
> ```text
> geneA1    geneB1
> geneA2    geneB2
> ```
> - Gene IDs in the homologous gene pair list must exactly match the gene identifiers in the GFF3 files, 
> using the `Name` attribute when available, or the `ID` attribute otherwise.
> - If parental CDS files are provided, the gene IDs in the CDS files must exactly correspond to those specified 
> in the homologous gene file. Homologous gene pairs sharing identical CDS sequences will be automatically 
> annotated as `Same`.


## Result
Genes are classified into one of the following five categories based on their similarity to the two parental genomes:
- **A**: The gene is more similar to the homologous gene in parent A.
- **B**: The gene is more similar to the homologous gene in parent B.
- **Same**: The homologous genes in parents A and B have identical CDS sequences.
- **Undetermined**: The gene cannot be confidently assigned to either parent A or parent B.
- **Missing**: The gene is not detected in the sample.
